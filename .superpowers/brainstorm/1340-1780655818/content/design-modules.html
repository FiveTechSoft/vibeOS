Diseño vibeOS — Módulos y Flujo de Datos

Enfoque A: Monolito Harbour. Sección 1 de 4.

1. Estructura de Archivos

vibeOS/
├── src/
│ ├── vibeos.prg ← main(): arranca servidor, monta rutas
│ ├── vibehtml.prg ← construye system prompt, llama a DeepSeek, parsea respuesta
│ ├── vibesession.prg ← windowSessions map, estado de ventanas, persistencia
│ ├── vibedelta.prg ← merge de deltas HTML, diff mínimo, IDs anidados
│ ├── vibestatic.prg ← sirve assets/ (CSS, JS, iconos)
│ └── vibeos.hbp ← Harbour build file
├── assets/
│ ├── xp.css ← Windows XP Luna theme CSS (librería completa)
│ ├── xp.js ← comportamientos: menús, tabs, tree-view, drag, resize
│ └── vibe.js ← glue: envía clicks/input al backend, aplica deltas HTML
├── ccharbour/ ← módulos reutilizados de CCHarbour (symlink o copia)
│ ├── ccapi.prg
│ ├── cchttp.prg
│ ├── ccsse.prg
│ └── ccconfig.prg
└── harbour/ ← libs del compilador Harbour
└── contrib/hbhttpd/ (linkado estáticamente)

2. Flujo de Datos — Arranque

1. Usuario lanza vibeos.exe
2. vibeos.prg:main() → inicia UHttpd en puerto 8765
3. Monta rutas:
  / → redirige a /desktop
  /desktop → vibehtml:DesktopInit() — llama a DeepSeek con system prompt → devuelve HTML inicial del escritorio XP
  /api/action → vibehtml:HandleAction() — recibe interacción del usuario → devuelve delta HTML
  /assets/* → vibestatic:Serve() — archivos estáticos
4. Navegador abre http://localhost:8765 → ve escritorio Windows XP
